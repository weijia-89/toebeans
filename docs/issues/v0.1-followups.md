## 2. Timezone onboarding + travel-mode toggle

**Source:** Test-as-spec review 2026-05-15, decision D6. ADR-0007 (stub).
**Slice:** 1.5 (after the phase-create UI exists; before "show schedule for next 30 days" lands).
**What:** Three things:
  - On first pet creation: onboarding screen explaining "your pet's reminders follow your phone's timezone. If you travel without your pet, mark that on the pet's profile."
  - On `Pet` entity: optional `homeTimezone` field. When set, materialization uses this TZ instead of `TimeZone.currentSystemDefault()`.
  - On `Schedule` entity: optional `travelMode` enum (`FOLLOW_PHONE` | `STAY_HOME_TZ` | `ELAPSED_INTERVAL`). Default `FOLLOW_PHONE`.
**Acceptance:** ADR-0007 promoted from PROPOSED to Accepted. New `SchedulePhaseDstRulesTest`. Integration test simulating `ACTION_TIMEZONE_CHANGED`.
**Why not v0.1:** Requires design work — see ADR-0007 "Open design choices."

## 3. Real DB lookup in `DoseAlarmReceiver`

**Source:** v0.1 hand-back item #5. `androidApp/src/main/kotlin/app/toebeans/android/notifications/DoseAlarmReceiver.kt` ships with a placeholder `ScheduledReminder`.
**Slice:** 1 (alongside the persistence layer).
**What:** Receiver looks up the firing `DoseEvent` by id from SQLDelight, builds a real `ScheduledReminder`, dispatches to the actuator. Handles the "row gone" race (user deleted the schedule between scheduling and firing) by silently cancelling.
**Acceptance:** Robolectric test for the happy path + the row-gone case.

## 5. PendingIntent collision mitigation

**Source:** Adversarial finding LOW (`@/Users/wjia/Projects/toebeans/androidApp/src/main/kotlin/app/toebeans/android/notifications/AndroidNotificationActuator.kt`).
**Slice:** 1.
**What:** Replace `reminderId.hashCode()` request-code derivation with a monotonic int counter persisted alongside the reminder. Eliminates the ~65k-reminder birthday-paradox collision.
**Acceptance:** Test that inserts two reminder ids with the same `hashCode()` (constructed via the `String.hashCode` collision space — easy to find) and asserts independent scheduling.

## 7. Argon2id key derivation for backup cipher

**Source:** `feat(backup)` commit + ADR sketch in `BackupCipher` KDoc.
**Slice:** 2.
**What:** Replace PBKDF2-HMAC-SHA256 (600k iterations) with Argon2id once a KMP-native (no JNI) implementation exists. Until then, PBKDF2 + the AES-GCM auth tag is the v1 posture.
**Acceptance:** New cipher version bump (envelope magic `TBN2`), backward-compat path for `TBN1` envelopes during decrypt.
